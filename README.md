# K뷰티 AI 제휴마케팅 사장실

초보자가 K뷰티 제휴마케팅을 시작하기 전에, 재고를 사지 않고 먼저 콘텐츠와 제휴 링크로 시장 반응을 검증하기 위한 단일 HTML 도구입니다.

## 바로 열 파일

`처음_열기.html`

이 파일 하나만 열면 됩니다.

## 무엇을 할 수 있나

| 기능 | 설명 |
|---|---|
| 초보자 마법사 | 다음에 뭘 해야 하는지 단계별로 안내 |
| StyleKorean 조건 확인 | 소셜 채널, 팔로워, 게시물, PayPal 준비 여부 체크 |
| COSRX 콘텐츠 원고 | 블로그 원고와 쇼츠 대본 제공 |
| 제휴 링크 삽입 | 실제 제휴 링크를 넣으면 원고와 성과 입력칸에 반영 |
| 발행 전 검수 | 제휴 고지, 위험 표현, 실제 링크 여부 확인 |
| 성과 추적 | 조회수, 클릭수, 구매수, 커미션을 기록하고 CTR/CVR 계산 |

## 사용 순서

1. `처음_열기.html` 파일을 브라우저로 엽니다.
2. 첫 화면의 `오늘의 결론`과 `다음 행동`만 봅니다.
3. `👉 지금 딱 하나` 대표 버튼을 누릅니다.
4. 열린 탭에서 필요한 입력이나 확인만 합니다.
5. 더 자세한 설명이 필요할 때만 초보자 마법사를 엽니다.

## 현재 사용법: 첫 화면만 보면 됨

상국님은 처음에 아래 순서만 보면 됩니다.

| 화면 위치 | 보면 되는 내용 | 상국님 행동 |
|---|---|---|
| 오늘의 결론 | 지금 사업 단계 | 재고를 사지 않고 반응 검증 |
| 다음 행동 | 지금 딱 하나 할 일 | 대표 버튼 클릭 |
| 상태판 | 가입, 상품, 콘텐츠, 성과 상태 | 부족한 상태만 확인 |
| AI 직원 카드 | 담당 업무별 바로가기 | 필요한 직원 카드 클릭 |
| 자세한 마법사 | 초보자용 단계 설명 | 막힐 때만 열기 |

## 초보자 퀘스트 안내 사용법
처음 화면이 막막하면 게임 튜토리얼처럼 `🎮 퀘스트 다시 보기` 버튼을 누르면 됩니다.

| 버튼/표시 | 의미 | 사용법 |
|---|---|---|
| 🎮 퀘스트 다시 보기 | 초보자 안내를 처음부터 다시 시작 | 길을 잃었을 때 누르기 |
| 다음 ▶ | 다음 단계로 이동 | 말풍선이 가리키는 곳만 보고 누르기 |
| 오늘은 숨기기 | 오늘은 안내를 닫음 | 이미 아는 내용이면 닫기 |
| 초록 테두리 | 지금 봐야 하는 화면 위치 | 이 영역만 확인하기 |
| 진행률 막대 | 퀘스트가 어디까지 왔는지 표시 | 100%가 되면 첫 안내 완료 |
| 상국님 할 일 / AI 직원 할 일 | 역할 구분 | 직접 할 일과 AI가 돕는 일을 나눠 보기 |
| 오늘 진짜 할 일 1개 | 퀘스트 완료 후 큰 버튼 | 다음 화면을 바로 열기 |
| 🏆 퀘스트 완료 | 첫 실행 안내 끝 | 첫 번째 오늘 할 일이 자동 체크되고 가입·링크 확인부터 진행 |

## 상국님이 직접 해야 하는 일

| 작업 | 이유 |
|---|---|
| StyleKorean 로그인/가입 | 계정 권한 필요 |

| 작업 | 설명 |
|---|---|
| 원고 작성 | COSRX 블로그/쇼츠 초안 제공 |
| 제휴 고지 확인 | “제휴 링크 포함” 문구 누락 방지 |
| 위험 표현 확인 | 치료, 완치, 즉시 효과 같은 표현 방지 |
| 성과 계산 | CTR, CVR 등 기본 지표 계산 |
| 다음 행동 추천 | 클릭/구매 결과에 따라 수정 또는 확장 판단 |

## 초보자용 역할 분담표

상국님은 아래처럼 나눠서 보면 됩니다.

| 구분 | 상국님이 직접 하는 일 | AI가 대신 도와주는 일 |
|---|---|---|
| 계정 준비 | StyleKorean, PayPal, SNS 계정 로그인 | 필요한 준비물과 체크리스트 정리 |
| 상품 선택 | 최종으로 테스트할 상품 승인 | 후보 상품, 제목, 콘텐츠 방향 제안 |
| 콘텐츠 작성 | 최종 발행 버튼 클릭 | 블로그 글, 쇼츠 대본, 공유 메시지 작성 |
| 링크 관리 | 실제 제휴 링크 복사 후 입력 | 링크가 원고에 들어갔는지 검수 |
| 성과 판단 | 숫자 입력과 최종 결정 | CTR, CVR, 커미션 계산과 다음 행동 추천 |
| 위험 관리 | 비밀번호, 인증번호, 결제 정보 직접 관리 | 과장 표현과 제휴 고지 누락 확인 |

처음에는 딱 3가지만 하면 됩니다.

| 순서 | 오늘 할 일 | 완료 기준 |
|---|---|---|
| 1 | StyleKorean 가입 조건 확인 | 채널, 게시물, PayPal 준비 여부 확인 |
| 2 | COSRX 제휴 링크 준비 | 실제 링크를 `처음_열기.html`에 입력 |
| 3 | 성과 숫자 기록 | 조회수, 클릭수, 구매수, 커미션 입력 |

## 현재 첫 테스트 상품

COSRX Low pH Good Morning Gel Cleanser

처음에는 여러 상품을 동시에 하지 않고, COSRX 클렌저 1개로 콘텐츠와 링크 반응을 확인하는 구조입니다.

## 중요한 주의사항

- 재고를 먼저 사지 않습니다.
- 제휴 링크가 생기기 전에는 콘텐츠를 발행하지 않는 것이 좋습니다.
- 비밀번호, PayPal 인증번호, 개인 인증 정보는 AI에게 공유하지 않습니다.
- 제품 효능을 과장하지 않습니다.
- 치료, 완치, 미백 보장, 즉시 효과 같은 표현은 사용하지 않습니다.

## CODEX/CODEL 영구 운영 규칙

CODEX/CODEL은 이 프로젝트에서 같은 운영 원칙을 의미합니다.
이 프로젝트에서 Codex는 아래 기준으로 계속 운영합니다.

| 항목 | 운영 규칙 |
|---|---|
| 보고 형식 | 항상 표 중심의 보고서 형식으로 정리 |
| 진행률 보고 | 긴 작업은 약 10초마다 `진행률 20%`, `진행률 45%`, `진행률 70%`처럼 현재 위치와 다음 행동을 짧게 보고 |
| 다음 작업 추천 | 작업 완료 후 반드시 `다음 우선순위 추천 작업` 1~3순위 제시 |
| 진행 방식 | 기본은 1순위부터 진행, 사용자가 `전체 진행`이라고 하면 1~3순위 모두 진행 |
| 초보자 기준 | 상국님은 완전 초보자 기준으로 설명 |
| 핵심 파일 | 실제 사용 파일은 `처음_열기.html` 하나로 유지 |
| 자동화 원칙 | 로그인, 결제, 인증, 최종 발행 외에는 AI가 최대한 처리 |
| 검증 원칙 | 수정 후 JavaScript 문법 검사와 가능한 브라우저 확인 수행 |
| Git 운영 | 요청 시 커밋과 푸시까지 직접 처리 |
| 위험 관리 | 비밀번호, PayPal 인증번호, 개인정보는 AI에게 공유하지 않음 |
| 표현 검수 | 치료, 완치, 즉시 효과, 미백 보장 같은 과장 표현 금지 |
| 사업 판단 | 조회수, 클릭수, 구매수, 커미션으로 성과 판단 |

핵심 문장:

> 상국님은 `처음_열기.html` 하나만 열고, 마법사 버튼을 따라가며, 돈 쓰기 전에 콘텐츠와 제휴 링크 성과를 먼저 검증한다.

진행률 보고 예시:

| 진행률 | 보고 방식 |
|---|---|
| 진행률 20% | 지금 무엇을 확인 중인지, 어디서 멈췄는지 보고 |
| 진행률 45% | 문제 원인이나 결정된 방향을 보고 |
| 진행률 70% | 마무리 전에 검증할 내용과 다음 행동을 보고 |

## 작업 요청 예시 문장

상국님은 아래 문장처럼 짧게 말해도 됩니다.

| 말하면 되는 문장 | Codex가 이해하는 뜻 |
|---|---|
| `1순위 진행해` | 다음 우선순위 추천 작업 중 1순위만 진행 |
| `1순위 2순위 3순위 모두 진행` | 추천 작업 3개를 순서대로 모두 진행 |
| `전체 진행` | 현재 제안된 작업을 가능한 범위에서 전부 진행 |
| `커밋 푸시해` | 변경사항을 GitHub 저장소에 커밋하고 푸시 |
| `HTML로 만들어줘` | 보고서나 자습서를 다운로드 가능한 HTML 파일로 제작 |
| `초보자용으로 다시 설명해` | 어려운 말 없이 처음 하는 사람 기준으로 다시 정리 |
| `다음은 뭐야?` | 지금 상태에서 가장 중요한 다음 행동을 1~3순위로 추천 |
| `필요 없는 파일 제거해` | 실제 사용하지 않는 파일을 정리하고 핵심 파일만 남김 |
